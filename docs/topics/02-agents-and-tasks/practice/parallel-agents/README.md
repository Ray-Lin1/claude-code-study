# 实践项目：并行 Agent 调用

## 目标

学习如何在 Claude Code 中并行调用多个 Agent 处理独立任务。

## 背景

你正在分析一个新项目，想了解不同方面的信息。这些查询是独立的，可以同时进行。

## 任务

使用并行 Agent 完成以下分析：

### 挑战 1：并行文件搜索

告诉 Claude Code：
```
"并行调用 3 个 Explore Agent，分别查找：
1. 项目中所有的 Python 测试文件
2. 所有的配置文件（.json, .yaml, .toml 等）
3. 所有的文档文件（.md）"
```

观察 Claude Code 如何在一次响应中调用多个 Task 工具。

### 挑战 2：并行代码分析

选择一个你熟悉的项目，让 Claude Code：
```
"使用并行 Agent 同时分析：
1. 项目的依赖关系
2. 主要的模块和功能
3. 代码的组织结构"
```

### 挑战 3：独立任务执行

创建 3 个独立的简单任务（比如查看 3 个不同文件的内容），让 Claude Code 并行处理。

## 学习要点

- 并行调用的语法
- 识别可以并行的任务
- 理解 Agent 之间的独立性
- 查看并行执行的结果汇总

## 预期结果

你应该能够：
1. 识别哪些任务可以并行执行
2. 正确请求并行 Agent 调用
3. 理解并行执行的效率提升
4. 看到多个 Agent 的结果汇总

## 对比实验

尝试串行和并行两种方式：
1. 串行：一个一个完成任务
2. 并行：同时完成任务

比较两者的时间差异（即使是小任务，也能看到概念上的区别）

## 进阶挑战

设计一个更复杂的并行场景：
- 需要使用不同类型的 Agent（如 Explore + Bash）
- 4 个以上的并行任务
