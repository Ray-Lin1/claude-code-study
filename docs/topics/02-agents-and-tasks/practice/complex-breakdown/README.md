# 实践项目：复杂问题拆解

## 目标

学习使用 Plan Agent 拆解复杂问题并逐步实现。

## 背景

你需要实现一个中等复杂度的功能，比如"为应用添加日志系统"。

## 任务

### 第 1 步：使用 Plan Agent 设计方案

告诉 Claude Code：
```
"使用 Plan Agent 设计一个日志系统的实现方案：
1. 需要哪些文件？
2. 使用什么日志库（Python logging？loguru？）
3. 日志级别和格式如何设置？
4. 如何配置日志输出到文件？
5. 考虑现有的项目结构"
```

### 第 2 步：审查计划

仔细阅读 Plan Agent 生成的方案，考虑：
- 是否覆盖了所有需求？
- 是否有遗漏的边界情况？
- 技术选择是否合理？

如有疑问，向 Claude Code 提问澄清。

### 第 3 步：执行实现

按照计划逐步实现：
1. 创建必要的文件
2. 实现核心功能
3. 编写测试
4. 文档说明

### 第 4 步：使用 Code-reviewer 验证

实现完成后，告诉 Claude Code：
```
"使用 Code-reviewer Agent 审查我刚刚实现的日志系统，
验证是否：
- 符合原始计划
- 遵循 Python 最佳实践
- 代码清晰易维护"
```

### 第 5 步：根据反馈改进

根据 Code-reviewer 的反馈进行改进。

## 学习要点

- Plan Agent 的使用时机
- 如何与 Plan Agent 交互（澄清、修改计划）
- 按计划执行的重要性
- Code-reviewer 的验证作用

## 预期结果

你应该能够：
1. 使用 Plan Agent 设计合理的实现方案
2. 理解计划中的权衡和选择
3. 按计划执行实现
4. 使用 Code-reviewer 验证工作
5. 根据反馈改进代码

## 复杂度选项

根据你的经验选择不同复杂度的任务：

**初级**：
- 添加一个简单的配置管理功能
- 实现一个工具函数库

**中级**：
- 实现日志系统
- 添加缓存层
- 实现 API 客户端

**高级**：
- 设计插件系统
- 实现异步任务队列
- 构建完整的功能模块

## 关键反思

完成后思考：
1. Plan Agent 的方案有哪些优点？
2. 有哪些地方可以改进？
3. 如果不用 Plan Agent，直接实现会有什么问题？
4. Code-reviewer 发现了哪些你没想到的问题？

## 进阶挑战

尝试完全自主地完成一个更复杂的功能：
1. 使用 Plan Agent 设计
2. 分阶段实现
3. 使用 Code-reviewer 验证每个阶段
4. 最后使用 Code-simplifier 优化
